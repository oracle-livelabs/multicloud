# Introduction

Oracle AI Database@AWS is a managed Oracle Database service that runs Oracle Cloud Infrastructure (OCI)–managed Oracle databases colocated inside AWS  data centers, giving AWS  applications low-latency access to Oracle Database services while billing and management are integrated through AWS.

Oracle Exadata Database Service on Dedicated Infrastructure on Oracle AI Database@AWS is Oracle’s high-performance database platform delivered natively inside AWS data centers as part of its multicloud offering. It brings the full power of Exadata—Oracle’s engineered system optimized for database workloads—closer to applications running on AWS.

Oracle Exadata Database Service on Dedicated Infrastructure on Oracle AI Database@AWS lets you run Oracle Exadata databases physically located in AWS, but managed using Oracle Cloud Infrastructure (OCI) services.

**Key points:**

Engineered for performance: Combines specialized compute, storage, and networking (RDMA, smart storage servers) to accelerate OLTP, analytics, and mixed workloads.

Low-latency integration with AWS apps: Since Oracle Exadata Database Service on Dedicated Infrastructure on Oracle AI Database@AWS is co-located in AWS regions, applications (e.g., on EC2/EKS) can access databases with single-digit millisecond latency over private networking.

OCI-managed experience: You use OCI tools/APIs for provisioning, scaling, patching, and backups, even though the hardware sits in AWS.

Private, secure connectivity: Integrated via ODB networking and peering—no public internet exposure.

Full Oracle Database feature set: Supports RAC, Data Guard and Exadata-specific optimizations.

Why it matters:
It removes the traditional trade-off between running apps in AWS and databases in OCI—giving you best-in-class Oracle database performance with tight integration into AWS-native architectures.

## About this Workshop
In this workshop, you learn how to create an ODB Network, Exadata Infrastructure, Exadata VM Cluster and finally an Oracle Exadata Database.

Estimated Workshop Time: 6 to 7 hour


### Objectives
In this workshop, you will create the following in AWS Console:

* An ODB Network
* An Exadata Infrastructure
* An Exadata VM Cluster

And you will create the following through OCI  Console:

* An Oracle Exadata Database


### Prerequisites
- Access to AWS Console, OCI tenancy, OCI console, and Oracle AI Database@AWS.
- An existing AWS Account. Your AWS account  must be onboarded and linked to an Oracle Cloud Infrastructure (OCI) tenancy.
- An Oracle Cloud Infrastructure (OCI) account with permissions to view and manage resources through the OCI console.

For detailed information about the prerequisites required to begin onboarding with `Oracle AI Database@AWS` and to create Oracle Exadata Database Service on Dedicated Infrastructure on `Oracle AI Database@AWS`, see [Prerequisites](https://docs.oracle.com/en-us/iaas/Content/database-at-aws/oaaws-prerequisites.htm).

You may now proceed to the next lab.
